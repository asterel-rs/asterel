# Code of Conduct

## Our standards

The Asterel project is for building a careful companion runtime. Keep discussion
specific, evidence-backed, and respectful of other contributors' time.

Expected behavior:

- Be direct without being hostile.
- Keep bug reports and reviews focused on reproducible behavior and concrete
  risk.
- Do not post secrets, private chat logs, private memory payloads, or personal
  identifiers.
- Respect coordinated disclosure for security issues.

Unacceptable behavior:

- Harassment, threats, targeted insults, or discriminatory language.
- Publishing another person's private information or credentials.
- Pressuring maintainers to run unsafe commands, expose secrets, or bypass the
  security process.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block users; or
escalate to GitHub moderation when needed to protect the project and its users.

Security vulnerabilities should be reported privately through GitHub Security
Advisories, not public issues.
