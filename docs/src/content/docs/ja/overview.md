多くの AI プロダクトは **ターン**、つまり送信した瞬間にモデルが返す単一の応答の品質を最適化します。Asterel は **ターンとターンの間に残り続けるもの**を最適化します。

これが全体の中心命題です。記憶バックエンド、感情トポロジー、共有ターンパイプライン、人格レイヤー、承認付きプランナーを主役にしない判断は、すべてここから出ています。

目指しているキャラクターは、意図的に静かです。普段は先導しない。けれど、ことばが出るときには重みがある。相手がまだ名づけていない輪郭を拾う。物腰は柔らかいが、判断はぼやけない。感情はあるが、それに飲まれない。

## 現在の状態

Asterel は初期段階のコンパニオン・ランタイムです。現在の中心は、Discord のテキスト会話、共有されたコンパニオン・ターンのパイプライン、記憶に支えられた継続性、そしてローカル運用者向けのガバナンス画面です。他のチャネルや一部の深いキャラクター機構は、README が現在の標準経路として示していない限り、alpha または設計上の到達点として読んでください。

## コンパニオンのループ

ランタイムは一回きりのリクエストと応答ではなく、ループを中心に作られています。

```text
会話する → 文脈を残す → 記憶を整理する → 距離感を調整する
         → 必要なときだけ再び入る → 関係が積み上がる
         → 創作や思考の伴走へ広がる
```

すべての設計判断はこのループに照らして評価されます。単一ターンを速くしても、その下の継続性を削るものは採用しません。

## 現在のプロダクト姿勢

Asterel は Discord 優先、テキスト優先です。コンパニオン・ランタイム、ゲートウェイ、記憶、人格、共有された事前補強経路が、現在のプロダクト検証の中心です。Discord テキストはプロダクトとして検証中の主チャネルであり、他のチャネルアダプターは README が別に示さない限り二次的、または alpha として扱います。デスクトップアプリはガバナンス、診断、記憶レビューのための運用者コンソールであり、ユーザーがコンパニオンと出会う主な場所ではありません。
